Getting Started

Registration, authentication and initial profile configuration

1

User Registration

Start by creating your account in MediNet:

  • Visit the main page of MediNet
  • Click on "Register" in the top right corner
  • Complete the form with your professional information
  • Confirm your email and complete the verification

Registration form with fields for professional and medical information

Login

Once registered, access the platform:

  • Use your email and password to access
  • The system will automatically redirect you to the dashboard
  • Optional: mark "Remember me" for persistent sessions

Project Management

Organize your medical research in collaborative projects

1

Create a New Project

Projects allow you to organize connections, datasets and models by research topic:

  • Click on "New Project" from the dashboard
  • Assign a descriptive name (e.g. "Cardiovascular Study 2024")
  • Add a detailed description of the objectives
  • Select a color for visual identification

Form to create a new medical research project

2

Project Management

Once created, you can manage your projects:

  • Change between projects: Use the project selector in the top bar
  • Edit project: Modify name, description or color
  • Delete project: Only if it has no associated connections
  • Collaborators: Invite other professionals (future function)

Management view with active projects list and edit options

connect_without_contact

Hospital Connections

Establish secure connections with other medical centers

1

Add New Connection

To participate in federated learning, you need to connect with other hospitals:

  • Go to "Datasets" → "Manage Connections"
  • Click on "Add New Connection"
  • Enter the connection data provided by the hospital
  • Assign the connection to a specific project

Required Fields:

business Name: Hospital identifier (e.g., "Hospital Clínic Barcelona")
computer IP: Server address (e.g. "192.168.1.100")
settings_ethernet Port: Communication port (e.g. "8080")
security Credentials: Username and password if required

Form to configure a new hospital connection

2

Validation and Test of Connections

Before using a connection, it is important to validate it:

  • Test Connection: Verify that you can communicate with the server
  • Validate Data: Check that the datasets are available
  • Status Monitoring: Monitor the status in real time
Connected - The connection is active and working
Connecting - Attempting to establish connection
Error - Connectivity or configuration problem

Status panel showing connectivity status of multiple hospitals

architecture

Model Designer

Create neural network architectures with visual interface

1

Designer Access

The visual designer allows you to create models without programming:

  • Go to "Models" → "Model Designer" from the main menu
  • Choose between creating a model from scratch or using a template
  • Select the problem type: classification, regression, etc.

Designer initial screen with template options and custom creation

2

Model Construction

Use the drag-and-drop interface to build your architecture:

input Input Layers

  • Input Layer - Define input dimensions
  • Reshape - Modify tensor shape
  • Flatten - Convert multidimensional data to 1D

layers Main Layers

  • Dense/Linear - Fully connected layers
  • Convolutional - For image processing
  • LSTM/GRU - For sequential data

tune Regularization

  • Dropout - Prevents overfitting
  • Batch Normalization - Stabilizes training
  • Pooling - Reduces dimensionality

Complete designer view showing layer palette, design canvas and configuration panel

model_training

Federated Training

Execute collaborative training between multiple institutions

1

Training Configuration

Configure specific federated learning parameters:

Federated Parameters

Rounds: Number of training rounds (typically 10-50)
Min Clients: Minimum hospitals to start training
Fraction Fit: Percentage of clients that train each round

Advanced Configuration

Strategy: Aggregation algorithm (FedAvg, FedProx, etc.)
Early Stopping: Early stopping criteria
Validation Split: Percentage for validation

Configuration panel for federated parameters and advanced options

2

Training Start

Once everything is configured, start the training process:

  • Review final configuration: model, datasets, parameters
  • Assign a descriptive name to the training job
  • Add an optional description for future reference
  • Click "Start Federated Training" to begin

Pre-training Validations:

check_circle Connectivity with all hospitals
check_circle Compatibility of selected datasets
check_circle Validity of model architecture
check_circle Availability of computational resources

Final view before starting training with summary and validations

Tips and Best Practices

Recommendations for best results

security

Security

  • Use strong and unique passwords
  • Regularly verify connection status
  • Do not share access credentials
  • Keep your profile information updated
speed

Performance

  • Start with simple models for testing
  • Optimize hyperparameters gradually
  • Monitor resource usage during training
  • Use early stopping to avoid overfitting
groups

Collaboration

  • Document your experiments in detail
  • Use descriptive names for models and projects
  • Share results with your team
  • Keep projects organized by topic
data_usage

Data Management

  • Verify dataset quality before training
  • Ensure data is balanced
  • Review compatibility between datasets from different hospitals
  • Use preview tools to understand the data
